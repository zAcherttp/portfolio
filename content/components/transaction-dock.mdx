## Usage

Mount the provider around the table or feed that opens transactions, then render one dock near the application root. A card begins expanded when opened. Swiping downward minimizes it; swiping a collapsed card upward restores it.

```tsx title="transaction-dock.tsx"
import {
  TransactionDock,
  TransactionDockProvider,
  useTransactionDock,
} from "@/components/registry/transaction-dock";

function TransactionTable() {
  const { openTransaction } = useTransactionDock();

  return transactions.map((transaction) => (
    <button
      key={transaction.id}
      type="button"
      onClick={() => openTransaction(transaction.id)}
    >
      View details
    </button>
  ));
}

<TransactionDockProvider transactions={transactions} maxExpanded={3}>
  <TransactionTable />
  <TransactionDock />
</TransactionDockProvider>;
```

## Stack behavior

- The least recently focused expanded card automatically collapses when capacity is reached.
- Manually minimized cards remain minimized until explicitly restored.
- Closing an expanded card restores the most recently auto-collapsed card.
- Collapsed cards share a deck slot so the dock remains inside the viewport.
- The provider reduces expanded capacity as the viewport narrows.
