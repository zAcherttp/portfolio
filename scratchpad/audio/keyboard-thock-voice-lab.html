How the reconstruction works

Key down loads the current project base: an immediate 60 ms smooth thump with five low modes, a low-passed noise transient, seed 100713, and no metal bank. Key up retains the fitted 78.4 ms preset with its strike at the recorded 34.5 ms crop position. No decoded sample data is copied into either procedural voice.

The central implementation is renderProcedural(). It returns a Web Audio AudioBuffer, so the sound can be played, changed, or exported without a server.

Auto fit remains available for Key up, where the recorded crop provides a target. Key down is manual-only until a key-down recording is supplied, so the optimizer cannot silently pull the thump toward the Up reference.
